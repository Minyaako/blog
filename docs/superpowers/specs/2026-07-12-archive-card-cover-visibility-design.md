# 归档卡片悬浮与封面展示控制设计

日期：2026-07-12

范围：归档卡片悬浮显示、列表卡片封面展示规则及相关自动化验证

## 背景

归档卡片的图片在鼠标悬浮时会轻微放大。当前图片容器没有裁剪放大后的溢出区域，而渐变遮罩仍受原始图片区域约束，因此原图颜色会从图片左边界漏入文字区域，形成明显的色块和错位。

目前列表卡片还把 `contentWarning` 与封面隐藏绑定在一起。视觉小说文章一旦启用内容提示，归档页便无法正常显示其页首图。这两个能力的目的不同：内容提示负责保护正文与摘要，封面展示则应由作者独立决定。

## 目标

1. 保留归档卡片现有的轻微图片放大动效，同时消除图片越界和渐变错位。
2. 默认在归档等文章列表中正常显示文章页首图，包括带内容提示的视觉小说文章。
3. 使用专用 Tag `隐藏封面` 控制列表卡片是否隐藏封面。
4. 保持文章内容提示、安全摘要和社交元数据的既有保护规则。

归档筛选、标签检索、页面进入动效及全站动效重构不属于本次范围。

## 方案选择

采用专用 Tag `隐藏封面`。它不需要扩展现有 frontmatter 结构，作者可以沿用 Git 与 Markdown 工作流直接控制展示结果，也符合当前“视觉小说封面是否显示依靠 Tag 控制”的要求。

未采用以下方案：

- 新增 `coverVisibility` 字段：数据语义更纯粹，但会增加内容模型与编辑界面约束，现阶段没有必要。
- 根据 `contentWarning` 自动隐藏：实现简单，但会继续耦合内容安全和视觉呈现，无法满足需求。

`隐藏封面` 是保留在文章 Tag 列表中的正式标签，会像其他标签一样显示在卡片和文章页中。这样作者可以直接看出隐藏原因，不增加不可见的特殊状态。

## 数据流与展示规则

列表卡片数据转换按以下顺序处理：

1. 文章没有配置 `cover` 时，保持现有无图卡片样式。
2. 文章配置了 `cover` 且 Tag 中不含精确值 `隐藏封面` 时，列表卡片输出并显示该封面。
3. 文章配置了 `cover` 且 Tag 中含精确值 `隐藏封面` 时，列表卡片不输出封面，使用现有无图样式。
4. Tag 匹配采用精确、区分大小写的中文字符串匹配，不进行模糊匹配或别名推断。

`contentWarning` 不再参与列表卡片封面的显示判断，但继续负责：

- 进入文章页时的确认弹窗；
- 未确认前的正文保护；
- 列表卡片中的安全摘要；
- RSS、搜索数据和社交元数据的既有敏感内容保护。

因此，当前视觉小说示例文章保留内容提示且不添加 `隐藏封面`，归档卡片会显示 `/images/posts/games-cover.webp`，同时仍显示安全摘要。文章页在确认内容提示后继续正常展示页首图。

## 归档卡片悬浮行为

图片区域成为独立的裁剪边界。图片只在该区域内缩放，不得进入左侧文字区域或越过卡片圆角。渐变遮罩继续覆盖图片区域并位于图片上方，其位置和尺寸不随图片缩放变化。

桌面端保留现有约 `1.025` 倍的轻微缩放与过渡时间；缩放中心保持在图片中央。移动端布局与现有断点不变。启用 `prefers-reduced-motion: reduce` 时继续遵守站点既有的减弱动效规则。

该修复不改变卡片宽高、文字区比例、图片裁切比例、圆角或颜色设计。

## 异常与兼容处理

- `隐藏封面` 与缺少封面的文章得到相同的无图布局，不渲染破损图片占位符。
- 封面路径失效时沿用浏览器与站点现有的资源错误行为，本次不新增客户端回退脚本。
- 旧文章无需迁移；只有作者主动添加 `隐藏封面` 才改变列表展示。
- Tag 只控制列表卡片封面，不移除文章页页首图，也不改变图片文件和版权信息。

## 验证

自动化测试覆盖：

1. 普通文章配置封面时，列表卡片数据保留封面。
2. 带 `contentWarning`、但不带 `隐藏封面` 的文章保留封面并继续使用安全摘要。
3. 带 `隐藏封面` 的文章不向列表卡片输出封面。
4. 当前视觉小说示例文章在归档页渲染正确的页首图。
5. 归档图片容器具有明确的溢出裁剪，悬浮后放大的图片仍限制在右侧图片区域内。

完成自动化验证后，在本地开发服务中检查桌面与窄屏归档页，重点观察卡片悬浮、圆角边界、文字区渐变和视觉小说封面。实现完成时保持本地服务运行，供浏览器复核。
